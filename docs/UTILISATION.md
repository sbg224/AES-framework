UTILISATION.md

AI Engineering System (AES)

Guide d’utilisation

⸻

1. Objectif

Ce document explique comment utiliser l’AI Engineering System (AES) dans le cadre du développement d’un projet.

Il décrit les principes d’utilisation quotidienne du framework.

⸻

2. Démarrer une nouvelle tâche

Avant toute intervention :

1. Prendre connaissance du contexte du projet (CONTEXT.md).
2. Comprendre l’architecture (ARCHITECTURE.md).
3. Identifier les technologies utilisées (STACK.md).
4. Consulter les décisions importantes (DECISIONS.md) si nécessaire.
5. Vérifier les audits ouverts (AUDIT.md).

⸻

3. Réaliser une tâche

Chaque tâche suit le processus défini dans WORKFLOW.md.

Les développements doivent respecter :

* les règles (RULES_OF_ENGAGEMENT.md) ;
* les standards (STANDARDS.md) ;
* l’architecture du projet.

⸻

4. Clôturer une tâche

La clôture d’une tâche suit l’Étape 9 de WORKFLOW.md : application de la CHECKLIST.md, vérification de l’état du dépôt, contrôle des données sensibles, puis validation explicite du développeur avant tout commit ou push.

⸻

5. Documentation vivante

Les documents suivants évoluent avec le projet :

* CONTEXT.md ;
* DECISIONS.md ;
* AUDIT.md ;
* LEARNING.md ;
* CHANGELOG.md.

Toute modification reste soumise à la validation du développeur.

⸻

6. Bonnes pratiques

Utiliser AES comme une référence permanente.

Les documents doivent être consultés lorsque leur contenu est pertinent, sans chercher à les mémoriser intégralement.

Le framework a pour objectif de centraliser les connaissances et de faciliter les décisions.

⸻

7. Références

Pour comprendre les principes qui fondent le framework, consulter PHILOSOPHY.md.

Pour installer AES dans un projet, consulter INSTALLATION.md.
